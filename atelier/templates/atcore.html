AtCore

AtCore is the API that all the work on serial communication. We work with the thinking were every firmware is a plugin where we handle the small differences between them.

Introduction

Inside AtCore we have a dictionary with all the GCode commands and that they do when called. So far we implemented methods that handle the most used GCode commands.

The below snnipet of code show one method that we use on logging, where every command sent has it's own string.

       QString GCode::toCommand(GCommands gcode, const QString &value1) {
              switch (gcode) {

                        return QStringLiteral("G0");

Test Client

For AtCore to work we need a client to control it. Since we need to test it, we developed, for now, a Test Client that we are using since the beggining. The Test Client implements all the basic features that we need to control the 3DPrinter. We are using it to validate the way that AtCore is implemented, while Atelier interface isn't ready.


Using Test Client

Connect

To use Test Client is simple, you can install it, and run. We have an automated whay to find the port that the 3DPrinter is using it. So connect your 3DPrinter to your PC and wait for the port to show it here:



Now you need to set the baud rate of your printer, make sure to set this otherwise the return of your printer will be misunderstood by AtCore and you will not be able to connect it. On my case it's 115200:



After that you can click on connect and check on the log or the status bar if the firmware is loaded correctly and if the printer is connected.



Temperature

For temperature control, you can change to the second tab, where you can set Bed and HotEnd temperatures, beyond the fan speed.



Temperature Plot

When everything is working, your printer will be checked every 5 seconds for bed and extruder temperatures and it will be show on the plot.



Axis Movement

For the move of the axis every piece of this pie have a default value to move the axis, if you pause the mouse on it, the tooltip will show you how much mm you can move. If you want to home one or all axis, you have buttons on your dispose to do it.



To Print

To print a GCode you need to click on Print File so you can load the GCode. When you click on 'Ok', the print job will start and a progress bar will be show below the log.



You have a few more features on the Print tab:

  • Emergency Stop: Obviously to Stop your printer.
  • On Pause: Based on the variety of printers, you can set your own Pause command, this is handy if you want to do a pause and change filament or do any thing that you want to. The On Pause action will work when you hit Pause Print button. You can send a lot of commands separated by a comma.

  • Control the Print speed and Flow.

Push GCode

Last but not least, on the Command tab you have tools to push a GCode command by hand and Show Message where you can push any message that you want to show on the LCD of your printer.



Now have fun! And give us feedback!
